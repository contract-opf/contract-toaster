# Test / development dependencies for contract-toaster.
#
# These are NOT runtime deps (see backend/requirements.txt) — they are only
# needed to run the test suite (tests/) and the local check gate
# (scripts/check.sh). Pinned so the gate is reproducible across machines and CI.
#
# Why each is here:
#   pytest      — runs the pytest-style test modules (e.g. tests/test_auth_jwt.py,
#                 tests/test_download_auth_attack.py). Every test file also has a
#                 __main__ script runner, but pytest is the canonical local runner.
#   jsonschema  — schema validation used by output-schema / playbook-lint tests
#                 (previously installed unpinned in CI GATE A).
#   boto3       — AWS SDK; imported by download.py and several worker/e2e tests.
#   moto        — mocks AWS services (S3/DynamoDB) so AWS-touching tests run
#                 deterministically offline (e.g. tests/test_retention_purge_worker.py,
#                 tests/test_orphan_reconciler_e2e.py, tests/test_review_submission_e2e.py).
#   python-docx — real-OOXML .docx parsing for scripts/build_anchor_map.py and
#                 scripts/diff_standard_form.py's --docx mode, and for
#                 scripts/generate_synthetic_standard_form.py. Elsewhere in the
#                 repo this is treated as an OPTIONAL dependency (gates degrade
#                 to SKIP, not FAIL, when it's absent — see
#                 tests/anchor/test_anchor_map_builder.py's docx-mode gate).
#                 It is declared here specifically so
#                 tests/diff/test_standard_form_loader_200.py (issue #200) is
#                 REQUIRED, not skipped, in this repo's own gate: that test's
#                 whole point is exercising real-docx mode against the
#                 SYNTHETIC placeholder standard form, so silently skipping it
#                 would silently stop covering the §10 sub-clause-split bug
#                 it was written to catch.
#
# boto3 and moto were previously undeclared test deps — the gate was not
# reproducible without them. Declaring + pinning them here fixes that.
